## A. Tujuan

1. Mendorong SMK untuk meningkatkan kualitas pelaksanaan Kegiatan Belajar Mengajar (KBM) yang mengacu pada Standar Keterampilan Kerja Nasional Indonesia (SKKNI) bidang keahlian Web Design Technology.
2. Mempromosikan keterampilan siswa SMK kepada dunia industri sebagai calon pengguna tenaga kerja.
3. Mempromosikan keterampilan siswa SMK bidang keahlian Graphic Design Technology kepada dunia industri.
4. Memberikan kesempatan dan motivasi kepada siswa untuk berkompetisi secara positif serta menumbuhkan kebanggaan terhadap bidang keahlian, sekolah, dan daerah.

---

## B. Lingkup Lomba

### 1. Tema

**Aplikasi Pemilihan Ketua OSIS Berbasis Web**

Jenis kegiatan berupa **tes praktik** dengan soal bersifat tertutup dan mengacu pada SKKNI Bidang Keahlian Desain Web, meliputi:

#### a. Modul 1: Frontend (Public & Voter)

- Pembuatan antarmuka publik
- Sistem voting siswa

#### b. Modul 2: Backend (Admin & API)

- Sistem manajemen data
- Penyediaan REST API

### 2. Teknik Penilaian

a. **Pengembangan Kode (Coding)**  
Menilai penulisan kode client-side dan server-side serta implementasi database.

b. **Manajemen & Keamanan Data**  
Menilai proses CRUD database dan kerapihan struktur direktori proyek.

c. **Pengujian & Debugging**  
Menilai proses testing untuk memastikan aplikasi bebas bug dan berjalan sesuai brief.

d. **Deploy & Dokumentasi API**  
Menilai kesiapan aplikasi di local server dan dokumentasi teknis REST API.

## C. Kisi-Kisi dan Kriteria Lomba

### 1. Modul 1: Frontend (Public & Voter)

a. Halaman publik dapat diakses tanpa login.  
b. Menampilkan rekapitulasi suara dalam bentuk angka, persentase, dan grafik informatif.  
c. Antarmuka voter dirancang mudah dan intuitif setelah login.  
d. Menggunakan HTML5, CSS3, dan JavaScript yang responsif dan interaktif.  
e. Diperbolehkan menggunakan library atau framework CSS/JS.

### 2. Modul 2: Backend (Admin & API)

a. Backend sebagai pusat pengelolaan data kandidat dan pemilih.  
b. Admin memiliki hak akses penuh (CRUD) dengan autentikasi aman.  
c. REST API menyediakan data dalam format JSON.  
d. Server-side programming (PHP / NodeJS) harus menjaga validitas suara.  
e. Internet hanya untuk dokumentasi resmi, **dilarang menggunakan AI** untuk generate kode.

---

## D. Lingkup Tugas Lomba

1. Memahami brief dan membuat sketsa alur logika aplikasi (flowchart/wireframe).
2. Coding dan implementasi database.
3. Testing aplikasi dan pengelolaan data digital yang rapi.
